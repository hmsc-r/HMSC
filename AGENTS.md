# HMSC Project Context

Hierarchical Modelling of Species Communities (**Hmsc**) is an R package providing a flexible framework for Joint Species Distribution Modelling (JSDMs). It relates species occurrences or abundances to environmental covariates, species traits, and phylogenetic relationships using a Bayesian framework with Gibbs Markov chain Monte Carlo (MCMC) sampling.

## Project Overview

- **Main Technologies:** R, MCMC (Gibbs Sampling), `coda`, `testthat`, `roxygen2`.
- **Core Functionality:**
    - Fitting JSDMs using `sampleMcmc`.
    - Defining model structures with `Hmsc` and `HmscRandomLevel`.
    - Predicting and evaluating model fit.
    - Handling spatial, temporal, and phylogenetic information.
- **Architecture:** The package follows the standard R package structure. Core logic is in `R/`, documentation in `man/` (generated by Roxygen2), and tests in `tests/`.

## Building and Running

This is a standard R package. The following commands are commonly used:

- **Install Dependencies:**
  ```R
  install.packages(c("devtools", "testthat", "roxygen2", "knitr", "rmarkdown", "coda"))
  # Plus other imports listed in the DESCRIPTION file
  ```
- **Document:**
  ```R
  devtools::document()
  ```
- **Build:**
  ```bash
  R CMD build .
  ```
- **Install:**
  ```bash
  R CMD INSTALL .
  ```
- **Test:**
  ```R
  devtools::test()
  # Or from terminal
  R CMD check .
  ```

## Development Conventions

- **R Package Standard:** Strictly adhere to CRAN-compatible R package structures and naming conventions.
- **Documentation:** Use **Roxygen2** for all function and class documentation in the `R/` directory.
- **Testing:** New features or bug fixes must include tests in `tests/testthat/`. Run `devtools::test()` to verify.
- **Coding Style:** Follow the existing style in the codebase (generally `camelCase` for functions, `Hmsc` class name).
- **Vignettes:** Documentation and tutorials are located in `vignettes/`.

## Key Files and Directories

- `R/`: Source code for all R functions.
- `man/`: Manual pages (automatically generated).
- `tests/testthat/`: Unit tests.
- `vignettes/`: Long-form documentation and tutorials.
- `DESCRIPTION`: Package metadata and dependencies.
- `NAMESPACE`: Exported/imported functions.
- `R/Hmsc.R`: Main class constructor.
- `R/sampleMcmc.R`: Core MCMC sampling implementation.
